# 前提の分類カテゴリ

前提を洗い出す際に、以下の8カテゴリで漏れなく確認する。

| カテゴリ | 検証すべき問い |
| --- | --- |
| 技術的実現性 | この技術/APIで本当に実現可能か |
| アーキテクチャ適合 | 既存の設計方針・ADRと矛盾しないか |
| スコープ | 想定している範囲は適切か（過大/過小ではないか） |
| 依存関係 | 他の機能・サービス・チームへの依存はないか |
| 制約 | パフォーマンス・セキュリティ・互換性の制約を見落としていないか |
| 運用 | デプロイ・監視・メンテナンスの考慮が必要か |
| 移行 | 既存データ・既存ユーザーへの影響はないか |
| 障害時の振る舞い | 失敗ケース・エッジケースの考慮があるか |
