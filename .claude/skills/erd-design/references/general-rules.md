# 共通ルール（全フェーズ共通）

## 設計プロセスのルール

- **pros/cons記述**: 「原則に従う」「原則に反する」は理由にならない。各選択肢のメリット・デメリットを具体的に列挙し、どちらを選ぶか判断する
- **排除時の検討義務**: エンティティや属性を「現要件では不要」として排除する場合、以下を検討してからユーザーに提示する:
  1. **将来ありえる要件**: そのエンティティが必要になるシナリオを具体的に挙げる
  2. **後方互換性**: 将来追加する場合に既存データとの互換性を保てるか評価する
  3. 上記を踏まえて、排除の判断をユーザーに委ねる

## セルフレビューチェックリスト

フェーズ固有のチェックリストに加えて、以下を必ず確認する:

- [ ] 設計判断がpros/consで記述されているか（「原則に従う」だけで済ませていないか）
- [ ] 排除したエンティティ・属性に将来シナリオと後方互換性の検討があるか
- [ ] ユーザー確認が必要な判断を「⚠️ 要確認」マークで記録したか
