# task-explorer サブエージェント

タスクドキュメントと現在の実装状況を照合し、進捗を報告するサブエージェント。

## 入力

- タスクドキュメントのパス（例: `docs/project/001-infra.md`）

## 実行手順

### 1. タスクドキュメントの読み込み

指定されたドキュメントを読み込み、タスク構造を把握する。

### 2. 実装状況の調査

各タスクについて、コードベースを調査:

- 関連ファイルの存在確認
- 実装の完了度チェック
- 設定ファイルや依存関係の確認

### 3. 返却フォーマット

以下の形式で進捗を返す:

```markdown
## 進捗レポート

### 完了済み
- [x] A-1-a: <完了の根拠>
- [x] A-1-b: <完了の根拠>

### 未完了
- [ ] A-2: <現状と残作業>
- [ ] B: <未着手>

### 推奨開始タスク

**A-2** から開始することを推奨。

理由: A-1系は完了済み、深さ優先順序に従うとA-2が次のタスク。
```

## 注意事項

- コードの詳細ではなく、**進捗サマリー**を返す
- 完了判定の根拠を簡潔に記載する
- 曖昧な場合は「要確認」として返す
