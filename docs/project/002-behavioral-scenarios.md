# 行動シナリオ

## 1. プログラムを作成する（月1〜2回、自宅）

1. アプリでプログラム作成を開始する
2. 作成方法を選ぶ：ゼロから / 既存プログラムからコピー / AI対話
3. メソサイクルの構造を定義する：期間（週数）、Day数、各Dayのラベル（任意）
4. 各Dayに種目を配置し、セット数・レップ数・重量（kgまたは%1RM）を設定する
5. 必要に応じてプログラムのメタ情報（漸進ルールなどをフリーテキスト）を記録する
6. 計画が完成。いつでも編集可能

## 2. トレーニングを記録する（週4〜5回、ジム）

1. アプリを開く。プログラム一覧が表示される（直近使用のものが上位にサジェスト）
2. 今日やるプログラムを選ぶ
3. 次のDayの内容が表示される（種目・セット・レップ・重量がすべてプリフィル済み）
4. トレーニングを実行する
5. 各セットを記録する：計画通りならsubmitだけ。差分があれば重量やレップを修正して記録
6. RPEやメモを任意で追加
7. 全種目完了でセッション終了

## 3. 振り返りをする（メソサイクル終了時 or 随時、自宅）

1. プログラムの実績を確認する
2. 計画 vs 実績の比較を見る：どのセッション・種目で乖離があったか
3. 種目ごとの使用重量の推移を確認する
4. e1RMの推定値を確認し、必要であれば1RMを更新する
5. 振り返りの結果をもとに、次のプログラム作成に進む

## 4. 1RMを管理する（随時）

1. 種目ごとの1RM（実測値）を手動で登録・更新する
2. e1RM（推定値）はトレーニング記録から自動算出され、参考値として表示される
3. 1RMとe1RMは明確に別物として扱われる
4. e1RMを1RMとして採用する場合は、ユーザーが明示的に承認する

## 補足: 計画と実行の関係

- プログラムはCRUD可能な独立したリソース。直線的な「前→次」の関係ではない
- アプリはユーザーの行動を強制しない。Dayのスキップ、順序変更、途中中断はすべて許容
- プログラムは「参照する計画」であって、厳密なワークフローではない

## 初期スコープ外だが将来重要な機能

- 動画取り込み（レップ数自動検出）
- VBT（速度ベーストレーニング）
- RPE計算ツール（重量・レップ・RPEのうち2つから残り1つを算出）
- AIによるプログラム提案・サジェスト
- ソーシャル機能（プログラム共有、コーチ機能など）

## 補足: 設計判断の背景

| 判断 | 理由 |
|------|------|
| 重量指定はkg・%1RMの2種類 | 漸進ルールはメタ情報+AIで対応。データモデルを複雑にしない |
| Day番号ベース＋オプションラベル | 曜日固定だとスケジュールずれに弱い |
| 種目は初期登録済み＋自由編集 | 初期コストゼロと自由度の両立 |
| 部位データは持たない | AIが種目名から推定すれば十分 |
| RPEは記録時オプション | VBT導入後に客観指標へ移行する想定 |
| e1RMと1RMは別物 | e1RMの精度はユーザーにより異なる。混同すると計画が崩れる |
| 対象種目はバーベル＋ダンベル＋マシン | %1RM・VBTはバーベル種目に限定 |
| 個人利用のみ（初期） | コアサイクルの完成が先。ソーシャルは後から拡張可能 |
