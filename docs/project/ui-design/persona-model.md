# ペルソナモデル

## 属性

- ソフトウェア開発者。仕事と家庭がある
- 筋トレ歴は中級〜上級。複数のトレーニングプログラムをこなしてきた経験がある
- 理想は週6回以上トレーニングしたいが、現実は仕事・家族・体調の都合で週4〜5回

## 現在の課題

- 計画はスプレッドシート、記録は自作アプリと、ツールが分断されている
- 1RMから%計算、RPEの組み合わせなど、スプレッドシートでの計画作成に時間がかかりすぎる
- 計画と記録の往復・重複入力が億劫

## 求めていること

- 計画と記録が1つのアプリに統合されている
- トレーニングプログラムの入力に最適化されたUIで計画を立てられる
- 記録は計画値がプリフィルされていて、submitするだけ
- データドリブンな管理（RPEの主観より速度のような客観指標を信頼する）

## 価値観・設計思想

- アプリは道具。ユーザーの行動を型にはめない
- データモデルはシンプルに保ち、判断が必要なところはAIに任せる
- 「ないと困る」機能だけ初期スコープに入れる
